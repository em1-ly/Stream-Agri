Authentication summary  
The authentication of the TIMB web services makes use of the DIGEST authorization parameter supplied within the header section of the web service being requested by the client.
i.e. <Authorization> <Digest username:md5(password)>
If authorization is denied for the web service being requested by the client an HTTP 401 (Authorization failed) message is relied back to the client.

Booking and Registration authorization
When accessing a booking and registration related web service, the client should use the user names and passwords for valid users who currently have credentials to access the booking and registration system. The web services to use these credentials include the:
    1. Grower web service
    2. Booking web service

Stop order System authorization
When accessing a stop order system, grades and matrix related web services, the client should use the user names and passwords for valid users who currently have credentials to access the stop order system. The web services to use these credentials include the:
    1. Bale file web service
    2. Deduction file web service
    3. Transporter file web service
    4. Stop order file web service
    5. Creditor web service
    6. Grades web service
